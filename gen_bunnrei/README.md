# 指導計画 文例コーパス（土台）

保育所保育指針・教育保育要領の公式枠組みに沿って、指導計画の文例データベースを
組合せ生成するツール一式。**監修前の土台**であって完成品ではない。

## 生成規模

| 計画種別 | 件数 |
|---|---|
| 月案 | 8,760 |
| 週案 | 5,520 |
| 個人案 | 2,928 |
| 年案 | 1,504 |
| **合計** | **18,712** |

少数の基底テンプレート（約60文型）を「計画種別 × 年齢/月齢 × 月/期 × 記入項目 ×
領域」の5軸で展開した結果。ベンダー文例集と同様、人手で書くのは基底文だけで
組合せ空間は機械展開で埋める構造。事前に推定した「数千〜数万件」の上振れ側に着地。

## ファイル

- `framework.py` — 分類軸・領域・ねらいの方向性。**指針告示由来**（再配布可）
- `templates.py` — 基底文例テンプレート。**AI生成の文面**（要監修）
- `generate.py` — 展開生成器。`python3 generate.py` で `bunrei.sqlite` を出力
- `models.py` — SQLModel 定義（FastAPI スタック向け）
- `bunrei.sqlite` — 生成済みDB（18,712件）
- `sample.jsonl` — 先頭40件サンプル

## 出所タグ（各行が2つの性質を持つ）

- `framework_source = shishin_framework`：分類軸・領域・ねらいの方向性は
  保育所保育指針（厚労省告示第117号）由来。**告示本体は著作権法第13条により
  著作権の目的とならない**ため、枠組み語彙はそのまま利用可。
- `text_provenance = ai_generated`：文例の**文面**は本ツールが生成した創作。
  指針に準拠させてはいるが、指針が保証する正答ではない。
- `needs_review = True`：全行が監修未了。本番投入前に保育士監修が必須。

## 何が「できて」何が「できていない」か

できている：5軸スキーマ、指針準拠の分類枠組み、数万件規模への展開、出所・監修
フラグ、年齢×季節×項目での絞り込みインデックス。

できていない（＝監修の領域）：
1. **発達段階の妥当性** — 月齢ごとのねらいが発達的に適切かは未検証。
2. **文章の自然さ** — 乳児3視点を「{視点}の力」と接続するなど、機械展開ゆえの
   不自然な言い回しが残る。これは over-polish せず残してある。文面の品質が
   コードではなく監修コストである、という構造を可視化するため。
3. **「生きた文章」** — 業界がAI生成に求める「自園の記録に基づく具体性」は、
   静的展開では出せない。第二世代（記録ベースAI生成）の領域。

## 著作権についての注意

- 指針・要領の**告示本体**は著作権対象外。枠組みに使用済み。
- 指針**解説**は別途利用条件の確認が必要（本ツールでは未使用）。
- 公開web文例サイト（保育士バンク等）の文面は各サイトに著作権が帰属するため
  **種データに使用していない**。本コーパスの文面は全て本ツールの生成物。

## 推奨する次工程

1. `text_provenance` に `curated`（監修済み）`facility`（自園入力）を追加運用。
2. 監修フローを承認ワークフローに接続（担任が候補選択→主任が文面監修→確定）。
3. 第二世代として、自園の保育記録から項目別ドラフトを生成し、同じ
   `Bunrei` 枠に `text_provenance=ai_generated, needs_review=True` で流し込む。
   第一世代のUI（項目フォーカスでのフィルタ表示）をそのまま再利用できる。
